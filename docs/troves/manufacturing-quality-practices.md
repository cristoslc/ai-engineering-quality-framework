## ISO 9001:2015 (Quality Management Systems)

The most widely recognized QMS standard. Seven core principles:

1. **Customer Focus** — Understand and meet current and future customer expectations
2. **Leadership** — Establish clear vision, shared objectives, management commitment
3. **Engagement of People** — Foster active and responsible participation
4. **Process Approach** — Manage activities as interrelated processes
5. **Improvement** — Systematically seek opportunities for improvement
6. **Evidence-Based Decision Making** — Use data and facts to support decisions
7. **Relationship Management** — Build partnerships with stakeholders

### QMS Components

Quality Planning → Quality Assurance → Quality Control → Quality Improvement (PDCA cycle)

### Key Tools

- Internal and external audits
- Corrective and preventive actions (CAPA)
- Document control
- Management review
- Process capability analysis

## Six Sigma

Data-driven methodology targeting 3.4 defects per million opportunities (DPMO).

### DMAIC Framework

1. **Define** — Quality objectives and project scope
2. **Measure** — Current process performance
3. **Analyze** — Root causes of defects
4. **Improve** — Solutions to eliminate root causes
5. **Control** — Sustain improvements through SPC, visual management, monitoring

### Key Tools

- Statistical Process Control (SPC)
- Failure Mode and Effects Analysis (FMEA)
- Design of Experiments (DOE)
- Pareto Analysis
- Ishikawa (Fishbone) Diagrams
- 5 Whys
- Process Capability Analysis (Cp, Cpk)

### Belt System

- Champion — executive sponsor
- Master Black Belt — coach and trainer
- Black Belt — full-time project leader
- Green Belt — part-time project participant

### ISO 9001 vs. Six Sigma

| Criterion | ISO 9001 | Six Sigma |
|-----------|----------|-----------|
| Nature | Certifiable QMS standard | Process improvement methodology |
| Objective | Meet requirements, standardize | Reduce variation, eliminate defects |
| Approach | Management system | Data-driven statistical analysis |
| Certification | External audit required | Personal certifications (Belt system) |
| Level | Macro (system structure) | Micro (specific processes) |

## Total Quality Management (TQM)

Organization-wide culture of continuous improvement.

### Core Principles

- Customer focus (customer decides quality)
- Total employee involvement (quality circles, suggestion systems)
- Process-centered (integrated system view)
- Strategic approach (quality as competitive advantage)
- Continual improvement (PDCA cycle)
- Fact-based decision making (SPC, data analysis)
- Communications (cross-functional teams)

### Key Insight

TQM states: "Customer decides quality, workers produce it, top management leads it." The key factor is to prevent errors, not detect them.

## Statistical Process Control (SPC)

Uses statistical methods to monitor and control processes.

### Key Concepts

- Control charts (X-bar, R, p, c, u charts)
- Western Electric / Nelson rules for anomaly detection
- 3σ (three standard deviations) threshold as standard
- Common cause vs. special cause variation
- Process capability indices (Cp, Cpk)
- Rational subgrouping for sampling

## Lean Manufacturing

Focuses on maximizing value while minimizing waste.

### Key Concepts

- **5S** — Sort, Set in order, Shine, Standardize, Sustain
- **Kaizen** — Continuous incremental improvement
- **Jidoka** — Automation with human intelligence (stop on abnormality)
- **Poka-Yoke** — Mistake-proofing (error prevention)
- **Andon** — Visual management system (stop-the-line)
- **Just-in-Time** — Pull-based production
- **Heijunka** — Production leveling
- **Gemba** — Go and see (observe the actual work)
- **Value Stream Mapping** — Identify waste in process flow

### Poka-Yoke Types

1. **Contact method** — Physical attributes detect errors
2. **Fixed-value method** — Count or sequence verification
3. **Motion-step method** — Process step verification

## Key Cross-Industry Insights

1. **Prevention over detection** — Both TQM and modern DevOps emphasize shifting quality left
2. **Statistical thinking** — SPC in manufacturing maps to monitoring/observability in software
3. **Standardization enables improvement** — You can't improve what you haven't standardized
4. **Human factors matter** — Poka-yoke and Andon recognize that humans make errors; systems should prevent them
5. **Continuous improvement is cultural** — Not a project, not a tool, but an organizational mindset
